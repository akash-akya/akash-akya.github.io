    View Source IP Locator

    Simple Plug based HTTP server for querying IP location

    This is a HTTP wrapper for Geo library.

    Sample server is deployed at ip-locator.fly.dev, check it out!

    curl -sLi ip-locator.fly.dev/location/180.219.213.135

    This server is loaded with provided sample dummy data.

    Documentation: https://akash-akya.github.io/docs/ip-locator

    endpoints

     Endpoints

    Server exposes single endpoint.

    • GET /location/<IP Address>

      Returns json encoded location data as response. Please check API spec in openapi.yaml for more details

    dependencies

     Dependencies

    • PostgreSQL

    • Elixir v1.14

    • Erlang/OTP v25.x

    development

     Development

    To get started you can use docker-compose run PostgreSQL which is a dependency for Geo

    # start database
$ docker-compose up

# create database
$ mix ecto.create -r Geo.Repo

# run migrations
$ mix ecto.migrate -r Geo.Repo

    Load sample dummy data

    $ mix geo.import samples/sample.csv

    Run the application server

    $ mix run --no-halt

    Test the API

    $ curl -si localhost:4001/location/180.219.213.135

    NOTE

    Not using Phoenix since as the requirement is simple.

    production-deployment

     Production Deployment

    Building the container

    Dockerfile is provided to build a the server as container to simplify the packaging and deployment. We use mix release to package the server which is then copied to a container which is easier to deploy

    Since Geo is in a private git repo we need to pass a whitelisted private key to be able to pull the code.

    docker build --build-arg IP_LOCATOR_PRIVATE_KEY="$(cat id_ed25519 | base64)" -t ip_locator .

    You can now push this container to docker registry of your choice and can run the container.

    Configuration

    Set following environment variables

    • DATABASE_URL : (mandatory) database URL string, example: ecto://USER:PASS@HOST/DATABASE

    • PORT : (optional) HTTP server port. Defaults to 4001

    Refer Geo docs for running the migrations.

    Metrics

    By default server emit multiple Prometheus metrics in production environment. Metrics will be available at port 9091 and path /metrics you can configure your server to collect metrics at this endpoint

    Importing Location Data

    There are multiple ways to import data in production

    • connect to production database locally and run the mix task, this is simpler approach with Platforms like Fly.io

    • copy csv file to production server and run the mix task in a production server. If you don't have access to mix task then you can use an approach similar to the one mentioned for database migration above

    Above approaches works but might be unsafe since we are exposing access to production database and server. Safer approach can be to run scheduled job and import data programattically either using any background job scheduler or using Kubernetes Job if you are already running Kubernetes.

    If you are deploying this server in Fly.io you can use check fly.toml configuration as reference. This is the config which used for running ip-locator.fly.dev server.
