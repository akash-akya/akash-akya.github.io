    Library to store and fetch IP Geo-location

    An interface between database to store and fetch IP location. Import IP location mapping using a CSV file and fetch location for given IP. Data is stored in a PostgreSQL database.

    dependencies

     Dependencies

    • PostgreSQL

    • Elixir v1.14

    • Erlang/OTP v25.x

    installation

     Installation

    Currently the library is in a private GitHub repository. So to use the library you should use the whitelisted deploy key or use your access-token repo.

    • ssh key pair is present in deploy_keys folder. Don't worry this key is generated and used only to access this repo

    • to generate token, go to https://github.com/settings/tokens and follow the steps

    def deps do

  {:geo, git: "git@github.com:akash-akya/geo.git", tag: "v0.1.0"}
  # if you are using token
  # {:geo, git: "https://<TOKEN>:@github.com/akash-akya/geo.git", tag: "v0.1.0"}
end

    Much simpler alternatively is to clone the git repo and use the file system path

    def deps do
  {:geo, path: "path/to/geo"}
end

    These slightly complicated steps won't be there if we publish the library to hex.pm.

    using-the-library

     Using the library

    Configure database credentials. See for (Ecto.Repo)[https://hexdocs.pm/ecto/Ecto.Repo.html] documentation for more details about the config

    config :geo, Geo.Repo,
  database: "ip_geolocation_dev",
  username: "postgres",
  password: "postgres",
  hostname: "localhost",

    Create database and run migration

    # create database
$ mix ecto.create

# run migrations
$ mix ecto.migrate

    Create csv file containing the IP Location data

    data.csv

    ip_address,country_code,country,city,latitude,longitude,mystery_value
200.106.141.15,SI,Nepal,DuBuquemouth,-84.87503094689836,7.206435933364332,7823011346

    Import data from a csv file using mix task

    # run `mix help geo.import` for more information
$ mix geo.import data.csv --verbose

    Alternatively you can import programmatically using Geo.batch_upsert/1 helper function

    Access mapping data using Geo.get_location/1 or using Geo.Repo

    $ iex -S mix
# using helper function
iex()> {:ok, location} = Geo.get_location("200.106.141.15")

# using Geo.Repo for more complex querying
iex()> import Ecto.Query, only: [from: 2]
iex()> locations = Ecto.Query.from(Geo.Location, limit: 10) |> Geo.Repo.all()

    For more detailed information please check the documentation

    production-deployment

     Production Deployment

    Running Migrations

    Since mix tasks won't be available in the mix release. A convenience helper module is provided to run the migrations. You can call this function to run the migration from your production environment

    # SSH to prod server and run
$ bin/my_app eval "Geo.Release.migrate"

    development

     Development

    To get started you can use docker-compose run PostgreSQL

    # start database
$ docker-compose up

# create database
$ mix ecto.create

# run migrations
$ mix ecto.migrate

    Load sample dummy data

    $ mix geo.import test/support/import_files/sample.csv

    running-tests

     Running Tests 

    # start database
$ docker-compose up

$ mix test --trace
