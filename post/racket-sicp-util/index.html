Import in Racket's sicp

2018-06-14
Comments

I recently started reading Structure and Interpretation of Computer Programs or sicp for short. It's wonderfully well written. can't recommend it enough. I'm using racket scheme for doing the exercises. because along with support for sicp it also has awesome picture language1. You can even just copy paste the code, It works without a any change. There is one caveat though. Some basic functions such as square are not defined in the language. So every time you need those functions you have to define it in that file. I needed functionality to import the files so that I can create utility.scm file containing basic functions, then import it whenever required. Racket has (require ...) but it doesn't work for sicp language. Turns out there is one more function for that, which works in sicp language.

In utility.scm

#lang sicp
(define (square x) (* x x))

(#%provide square)

then to import utility.scm

#lang sicp
(#%require "utility.scm") ;; Absolute or relative path

(display (square 10))

Note that both should be of same language (#lang sicp). Some exercises use the functions defined in the previous exercise, we can import them the same way.

I also wanted some kind of simple 'test framework' to make sure nothing is breaking. Its also helpful to understand the code if I revisit the exercise later as the input and output will be there and I don't have to struggle to create correct input-output!

For creating test framework functions (something like 'assert') you need a way to delay the execution. This needed because for logging we need the caller function along with arguments exactly as its called. If we write assert as normal function, arguments will be resolved before making a call to assert. see Applicative order & Normal order.

So I have two options to delay the execution

  • Wrap the call in a lambda & unwrap it the assert function
  • Write a macro!

The first option will work but you have to wrap all of the arguments in a lambda everytime you call assert function. meh.

Now we are left at the mercy of macros! good news is its very simple. Thanks to awesome documentation. But there is a catch. Since we aren't using #lang racket, we can not use macro constructs. Solution? just import racket/base!

(#%require racket/base)

Now you can export the test function same as before

In utility.scm

(define (success lhs rhs)
  (display "Passed..."))

(define (failure lhs rhs)
  (display "Failed!  ")
  (display lhs)
  (display "  =  ")
  (display rhs))

(define-syntax-rule (test-equal lhs rhs)
  (let ((qlhs (quote lhs))
        (qrhs (quote rhs)))
    (cond [(equal? lhs rhs) (success qlhs qrhs)]
          [else (failure qlhs qrhs)])
    (newline)))

Now I can use test-equal similar to assert

(test-equal (+ 1 2 3) 6)
